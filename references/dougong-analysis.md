# 斗拱细部复原分析

## 参考图关键观察（dougong-bracket-detail.jpg）

### 整体尺寸
- **总宽度**: 1881.0 mm (约114分 @ 16.5mm/fen)
- **基础宽度**: 1155.0 mm (约70分)  
- **深度**: 相似，多层向上堆叠

### 四层结构分解

#### 第1层（底部基础）
- 宽度: 1155.0 mm
- 材料: 浅色（木制基础块）
- 作用: 栌斗 (Ludou) 基座

#### 第2层（下层斗拱）
- 特征: 下部有弧线卡榫，上部方形斗体
- 宽度: ~1400 mm（向外扩展 ~122.5mm）
- 构件: 华栱下跳 (Hua-gong jump 1)

#### 第3层（中层斗拱）
- 特征: 继续向上，宽度进一步收窄
- 中央出现独立支撑柱（深红色竖向构件）
- 宽度: ~1300 mm
- 构件: 华栱上跳 (Hua-gong jump 2) + 中央柱

#### 第4层（顶部）
- 最窄层，高度最小
- 中央支撑柱延伸到此
- 宽度: ~1150 mm（回收到接近基础）
- 构件: 令栱 (Ling gong) 或顶部斗

### 中央支撑结构（关键细节）
- **竖向穿插**: 从第2层开始，中央有竖向构件贯穿上升
- **定位**: 正好在中心轴线上
- **宽度**: ~70-100mm
- **高度**: 跨越多层，接近斗拱总高的60-70%

## 现有模型 vs 参考图

| 特征 | 现有模型 | 参考图 | 差异 |
|------|--------|-------|------|
| 层数 | 2层（ludou + hg1/hg2） | 4层清晰梯度 | **缺少中间层过渡** |
| 中央支撑 | 无明显竖向构件 | 贯穿式中央柱 | **缺少贯穿式支撑** |
| 水平宽度梯度 | 线性扩展 | 分段阶跃式 | **梯度关系不匹配** |
| 卡榫细节 | 简化为 box | 显示弧线咬合 | **细节过度简化** |

## 复原方案

### 增加中央支撑柱
```json
{
  "id": "puzuo-center-zhuzhu",
  "name_zh": "斗拱中央支撑柱",
  "name_en": "Central support post (中央柱)",
  "role": "Vertical post threading through bracket layers, concentrating load to column",
  "geometry": {
    "type": "cylinder",
    "r": 35,          // 约70mm直径
    "h": 150          // 跨越多层
  },
  "position": [0, 250, 0],  // 中心，从ludou顶部向上
  "provenance": "reference"
}
```

### 增加第2.5层、第3层的阶跃过渡
- 在现有 hg2 和顶部之间增加中间斗体
- 定义尺寸的梯度关系而非线性

### 详细化弧线卡榫
- 用更细致的几何替代简单的box
- 或至少在尺寸关系上体现弧线的作用

---

## 实施优先级

1. **高**: 添加中央竖向支撑柱（缺失最明显）
2. **高**: 调整水平尺寸梯度（hg1 → hg2 → 顶部的宽度递进）
3. **中**: 增加第3层的过渡斗体
4. **低**: 精细化弧线卡榫几何（可后续）

---

## 建模步骤

1. 在 structural-spec.json 中找到各 puzuo 组件
2. 为每个柱头铺作（pz-2, pz-3, ... pz-8）添加：
   - `puzuo-center-zhuzhu`: 中央柱，高度 ~150fen
   - 调整 hg1 和 hg2 的宽度，形成梯度
3. 可选：为柱头铺作顶部增加 `lingzhu` (令柱) 过渡层
4. 验证：视觉上应该看到从宽大基座逐级收窄到中轴线的效果
